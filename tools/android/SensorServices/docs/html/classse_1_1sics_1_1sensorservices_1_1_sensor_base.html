Detailed Description

A base class that can be extended in order to create a new sensor. This class includes all methods required for communication with a SensorsManager. To create a new sensor, you should extend this class:

public class MyCustomSensor extends SensorBase{};

In your sensor's constructor, you must call the superclass's constructor:

public MyCustomSensor(Params params){
    super();
}

Your custom sensor class will be sent several events by its parent SensorsManager during its lifetime. These are:

  • SensorBase#connect(SensorsManager)
  • SensorBase#startRecording()
  • SensorBase#stopRecording()
  • SensorBase#disconnect()

The basic principle is that you place code in SensorBase#connect(SensorsManager) that handles connecting to the sensor, and code in SensorBase#disconnect() that handles disconnecting from the sensor.

You may also place code in SensorBase#startRecording() and SensorBase#stopRecording() that handles starting and stopping of data recording respectively. For example, if your sensor generates some kind of summary report at the end of a recording session, this should be done in SensorBase#stopRecording().

Finally, your sensor may send and receive "new lap" and "summary" events. For example, your sensor can call SensorsManager#newLap() to indicate a new lap has occurred, and the event SensorBase#newLapDetected() may be sent to your sensor at any time. See the documentation of these methods for details on how to use them.

This class, and any subclass derived from it, is Serializable, so that sensor objects may be stored in Handler messages or Bundles.

void se.sics.sensorservices.SensorBase.connect ( SensorsManager  sensorsManager)

Connects to this sensor, and also receives a reference to the SensorsManager that is managing this sensor, which this sensor may use to obtain information from other sensors. You MUST create a method in your sensor class that overrides this method, and place any code you need to run to connect to the sensor in your method. void se.sics.sensorservices.SensorBase.newLapDetected ( )

Called by the parent SensorsManager when a new lap has been detected by one of the sensors. If you want your sensor to collect data lap-by-lap (for example the average heart rate during the last lap), then you should override this method and do your calculations in you new method. If you want to make the results of your calculations available for other sensors to use, you can publish it using SensorManager#storeLapValue(String, Object). You should NOT check for new lap data manually inside this method, as it is not guaranteed that all sensors will have published their data. Instead, once all sensors have had the chance to publish data, another method, SensorBase#newLapDataAvailable(HashMap), will be called in each sensor. This will provide the sensor with a HashMap containing any data published by any sensor during the last lap update.
