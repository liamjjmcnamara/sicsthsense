Detailed Description

A base Service that handles communication with a SensorsManager. You may extend this class in order to create a service that records data from sensors. This class takes care of all the basic service-related stuff, such as optionally acquiring a WakeLock. In your extended service, you should override Service#onStartCommand(Intent, int, int) and Service#onDestroy(), and place code in these two events for starting and stopping your service. Note that you must call super.onStartCommand() at the end of your own onStartCommand(), and super.onDestroy() at the end of your own onDestroy(). Additionally, you should call RecordDataService#stopRecording() at some point during your onDestroy().

void se.sics.sensorservices.RecordDataService.setUseWakeLock ( boolean  enableWakelock)
protected

Used to set whether or not this service should use a CPU wakelock to prevent the device from sleeping whilst the track is recording. Setting this to true will increase battery usage, but will ensure the service is not stopped by the operating system. The increase in battery usage may not be that great, as the track will most likely require constant GPS and accelerometer updates, as well as possibly continuous communciation with a Bluetooth sensor, all of which will reduce the time that the CPU is able to sleep.

Parameters
enableWakelockWhether or not the device should be prevented from sleeping
